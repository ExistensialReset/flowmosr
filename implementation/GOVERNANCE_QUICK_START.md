## PURPOSE

This Quick Start guide provides a **concise, practical overview** of Flow’s governance system for new participants and Node/Circle members. It explains **core principles, roles, and mechanisms**, enabling immediate participation while preserving system integrity.

---

## 1. CORE PRINCIPLES (TIER 1)

1. **Random Selection**  
   - Decisions are made by randomly selected participants (LOTUS panels).  
   - Panel size scales for statistical representativity and anti-manipulation (example: ≥1,000 for standard node).  

2. **Rotation & Delegation**  
   - Roles (mediators, panel members) rotate regularly.  
   - Delegation is temporary and transparent.  

3. **Pattern Detection & Reflection**  
   - System monitors for misclassification, hoarding, and bad-faith challenges.  
   - Threshold: ≥3 instances within 6 months triggers **reflection**, not punishment.  

4. **Conflict Metabolization**  
   - Conflicts are handled by **trained mediators** (rotating role).  
   - Mediation training is open, documented, and self-study friendly.  

5. **Soft Pressure Acknowledgment Rule (SPAR)**  
   - Any participant can pause discussions if social pressure is detected.  
   - Usage is documented **anonymously** to track systemic patterns.  

6. **Outcome Challenge Protocol (OCP)**  
   - If a decision seems misaligned, participants may challenge via LOTUS review panels.  
   - Same pattern detection rules apply: ≥3 ungrounded challenges → reflection.  

7. **Experimentation Requirement**  
   - Nodes are encouraged to run small, reversible experiments for learning and adaptation.  
   - Participation is flexible and voluntary, documented, and evaluated.  

8. **Privacy & Transparency Balance**  
   - Observations and data are anonymized where possible.  
   - Transparency focuses on **system health**, not individual surveillance.

## 2. GETTING STARTED

### Step 1: Join a LOTUS Panel
- Panels are randomly selected from Node participants.  
- Panel roles rotate automatically; no permanent committees exist.  
- Participation is documented anonymously for accountability and feedback.

### Step 2: Understand Your Role
- **Proposal submitter:** Defines initial classification (General / Hybrid / Technical).  
- **Micro-Panel:** Verifies classification and provides oversight.  
- **Mediator:** Facilitates conflict resolution; trained through open, rotating program.  
- **Participant:** Can raise SPAR or OCP flags as needed.

### Step 3: Participate in Decisions
1. Submit proposals following the random selection process.  
2. Verify classification if assigned to Micro-Panel.  
3. Engage in deliberation; raise SPAR if social pressure is detected.  
4. Participate in outcome challenges responsibly.  

### Step 4: Conduct Node Experiments
- Propose small, reversible experiments to improve systemic adaptation.  
- Document and share results for learning.  
- Optional: coordinate with Circle-level initiatives, but per-node experimentation is sufficient.

## 3. QUICK REFERENCE: KEY TOOLS

| Tool | Purpose | When to Use |
|------|--------|-------------|
| **LOTUS Panel** | Randomized decision-making | Any proposal requiring review |
| **Micro-Panel Verification** | Checks classification & prevents bias | For technical or hybrid proposals |
| **SPAR** | Pause discussion under social pressure | When any participant feels pressured |
| **OCP** | Review outcome misalignment | When a decision feels misaligned |
| **Reflection Protocol** | System-level learning | Triggered by repeated patterns (misclassification, hoarding, bad-faith challenges) |
| **Experimentation** | Improve system adaptation | Voluntary, reversible, node-level |

---

## 4. BEST PRACTICES

- **Document Everything:** Use Node or Circle logs for experiments, challenges, and reflections.  
- **Rotate & Share Roles:** Prevent power concentration.  
- **Use Reflection Over Punishment:** Treat deviations as data, not violations.  
- **Prioritize Anonymity:** Focus on systemic patterns rather than individual performance.  
- **Encourage Participation:** All members can act within their roles; everyone contributes to governance health.
