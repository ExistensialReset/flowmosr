# FLOW_SURPLUS_PROTOCOL.md
## Version 1.0
## Status: Active Structural Rule
## Scope: Global – Applies to All Nodes

---

# PURPOSE

This document defines how resource surplus above Baseline is:

1. Logged
2. Accounted for
3. Partially redistributed
4. Protected from coercive accumulation

This prevents Mammon-Logic re-emergence.

---

# I. DEFINITION OF SURPLUS

Surplus = Any resource held by an individual or Node that exceeds Global Baseline requirements per person.

Categories:

- Energy surplus (kWh above baseline)
- Space surplus (m² above 15 m² per individual)
- Water surplus (above 100 L/day per person average)
- Financial or exchange surplus (if hybrid currency exists)
- Infrastructure control surplus

Baseline is untouchable.
Surplus is accountable.

---

# II. LOGGING MECHANISM (FLOW LEDGER)

All surplus is recorded in:

Flow Ledger (Node-level → Global Aggregate Layer)

Ledger tracks:

- Quantity
- Duration held
- Voluntary allocation decisions
- Redistribution contributions

Personal privacy preserved at micro level.
System transparency preserved at macro level.

---

# III. REDISTRIBUTION THRESHOLD

Automatic redistribution activates when:

Surplus exceeds 2× Baseline in any resource category
AND
Another Node within region falls below Baseline.

---

# IV. REDISTRIBUTION RATE

Standard Rate:

30% of surplus above 2× Baseline
is allocated to:

- Baseline recovery pool
- Infrastructure stabilization pool
- Emergency Node buffer

70% remains under holder control.

This prevents hoarding without punishing productivity.

---

# V. INCENTIVE STRUCTURE

Surplus contributors receive:

- Flow Stability Credits
- Governance weight multipliers (temporary, capped)
- Priority in infrastructure allocation

Credits expire.
Power cannot permanently accumulate.

---

# VI. ANTI-ACCUMULATION SAFEGUARD

No individual or Node may:

- Control >10× Baseline resource equivalent without active review
- Withhold redistribution during verified deficit events

Violation triggers:

LOTUS review → Supermajority intervention → Forced balancing

---

# VII. VOLUNTARY ALLOCATION OPTION

Before automatic redistribution:

Holder has 7-day voluntary allocation window.

They may choose:

- Specific Node to support
- Infrastructure investment
- Energy grid expansion
- Water resilience system

If unused → automatic distribution activates.

---

# VIII. TRANSPARENCY RULE

Aggregate surplus statistics are public.

Individual-level data:

- Private by default
- Public if governance role > threshold

---

# IX. STRUCTURAL PRINCIPLE

Productivity is allowed.
Hoarding is not.

Growth is allowed.
Extraction without reciprocity is not.

Surplus exists to stabilize Flow.
