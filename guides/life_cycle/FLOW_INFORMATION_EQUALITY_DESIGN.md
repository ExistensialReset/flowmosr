# FLOW INFORMATION EQUALITY DESIGN (IED)

## 1. DELAYED RULE FEEDBACK
System thresholds (CRISIS, RESOURCE shifts, FMGL signals) are not immediately interpretable.

→ Actors only see outcomes after decisions, not triggers in real time.

---

## 2. RANDOMIZED TIMING WINDOWS
Key evaluations (RESOURCE simulation, CEPP review, FMGL updates) occur within randomized time intervals.

→ Prevents timing-based exploitation.

---

## 3. SYMMETRIC INFORMATION ACCESS
All participants receive the same information about:
- rules
- thresholds
- process states

But NOT:
- internal system weighting
- hidden proximity to triggers

---

## 4. DECISION BATCHING
Instead of continuous resolution:
- decisions are grouped in batches

→ reduces ability to react to single-event optimization

---

## 5. OBSERVABILITY PARITY RULE
If any actor can infer system state better than others:
→ system introduces noise or abstraction to restore parity

Not by hiding truth — but by preventing predictive asymmetry
