# FLOW CONFLICT RESOLUTION LAYER (FCRL)

## 1. PRIORITY ORDER (always applies in conflict)
When protocols disagree, apply this order:

1. **Baseline Survival**
2. **CRISIS Protocol (only if active crisis conditions are met)**
3. **SHARED_RESOURCE_CONFLICT**
4. **CEPP (autonomy and exit rights)**

## 2. TEMPORAL OVERRIDE RULE
Higher-priority layers may temporarily override lower ones, but:
- must be time-limited
- must be logged
- must revert automatically after conditions normalize

## 3. NO NEW RULES UNDER CONFLICT
During active conflict:
- no new protocols may be created
- no new exceptions may be introduced
- no structural changes may be made

Only resolution, not expansion, is allowed.

## 4. TIE-BREAK PRINCIPLE
If interpretation is unclear:
- choose the option that preserves **Baseline + minimal harm**
- if still unclear → choose simplest reversible action

## 5. FINAL INTERPRETATION LAYER
If all rules still conflict:
- humans decide using context and intent
- no protocol has absolute authority over interpretation
