## 3. Flow Node Integration (Simple)

### Option A — Individual Mode
- Keep UBIS personally  
- Voluntarily contribute time/resources  
- Node operates hybrid

### Option B — Pooling Mode (Recommended)
- Circle decides percentage of UBIS to pool  
- Node uses for collective infrastructure (land, food, housing, energy, tools)  
- Remaining UBIS is personal

### Option C — Full Commons Mode
- Pool 100% UBIS  
- Personal needs met through Node  
- High-trust, mature Nodes

**Rule:** Pooling is always voluntary. Exit is always possible.

**Flow Governance:**
- Circles (5-30 people)

**GCCS Governance:**

## 6. Onboarding for New Participants

1. **Learn the basics:** Flow Circle + Node concepts  
2. **Join or start a Node:** 5–50 people weekly  
3. **Ratify GCCS Charter digitally** (optional but recommended)  
4. **Decide pooling mode:** voluntary, transparent  
5. **Follow Flow protocols:** LOTUS governance, conflict mediation, care infrastructure  
6. **Experiment locally:** collect data, iterate, fork if needed  

## Your First Week (Concrete Steps)

**Day 1-2: Learn**
- Read: ARRIVAL.md (Flow)
- Read: GCCS Strategy page

**Day 3-4: Find people**
- Post on local community boards
- Reddit: r/Solarpunk, r/intentionalcommunity
- Ask: "Anyone interested in starting a Flow Circle?"

**Day 5-7: First meeting**
- Gather 5-10 people
- Location: Someone's home, park, library
- Agenda:
  1. Introduce Flow + GCCS (10 min)
  2. Share why each person came (20 min)
  3. Decide: weekly meetings? (10 min)
  4. First practical step: shared meal next week

**Week 2+: Build habits**
- Weekly meetings (same day/time)
- Try one Flow protocol (e.g., talking piece)
- Discuss: Would pooling UBIS help us? (when GCCS launches)
