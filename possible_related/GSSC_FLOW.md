## 🌍 Introduction: What is GCCS?

**Global Commons Capital System (GCCS)** is a macro-economic redistribution framework aiming to provide:

- **Universal Dividend:** unconditional basic income globally  
- **Commons Capture:** land, carbon, and natural resources managed as shared assets  
- **Legal Framework:** subsidiarity to protect local autonomy  
- **Resource Redistribution:** without centralized control

**Relationship to Flow:**  

- GCCS = “Here’s money for survival”  
- Flow = “Here’s how to live together without killing each other”  

**Principle:** Interoperability, not dependency. Flow nodes can operate fully without GCCS, but GCCS can accelerate scaling and financial resilience.

## 🏗️ Layered Model

LAYERED ARCHITECTURE
```
MACRO (GCCS) - Infrastructure
           |
    INTERFACE - Flow Node integration
           |
    MICRO (Flow) - Local execution, community governance
```

**Key Points:**

2. Both systems solve different problems. Neither depends on the other.

## 🔗 Practical Interface: How Money Moves

**Assumption:** GCCS provides ~$X/month/person globally  

**Flow Node Integration Example:**

1. Individuals receive dividend directly (personal account)  
2. Node proposes pooling agreement (optional, transparent)  
3. Participants vote (LOTUS or Circle decision)  
4. If pooled:
   - Funds → shared Node account  
   - Allocated via Node's economic protocol  
   - Individuals retain veto / exit rights  

**Example Scenario:**

- Sarah receives $500 GCCS dividend  
- Node proposes pooling 70% ($350)  
- She agrees (keeps $150 personal)  
- Node uses pooled funds for: land lease, food systems, infrastructure

## ⚠️ Failure Mode: GCCS Centralization / Capture

**Scenario:** GCCS becomes centralized or authoritarian  

**Flow Response:**

1. Dividend is helpful but **not required**  
2. Nodes can operate on hybrid/local funding  
3. Fork rights protect local autonomy  
4. Exit remains possible  

**Principle:** Flow remains resilient even if GCCS fails. Sovereignty is never compromised.
